## Story

As an operator concerned with LLM 成本失控,
I want `tally_llm_cost_cny_total{tenant,model}` + `tally_llm_tokens_total{tenant,model,direction}` Prom counter exposed at `/internal/v1/metrics`,
so that 「F1.4 AI Drawer ¥3.6/租户/月」之类的 plan 假设能被 Grafana 实测验证，而不是凭口算。

## Acceptance Criteria

1. 新包 `internal/pkg/llmgateway/metrics.go` 暴露两个 counter：
   - `tally_llm_cost_cny_total{tenant,model}` — 按 token×单价累加 CNY 元（float64 → counter, 保留 6 位小数）
   - `tally_llm_tokens_total{tenant,model,direction="in|out"}` — token 数累加
2. `internal/pkg/llmclient/client.go` 在解析完 `ChatResponse.Usage` 后调用 `llmgateway.RecordUsage(ctx, model, in, out)`；tenant 从 ctx 取（middleware 已注入 `tenant_id`）。
3. 单价表 hard-coded in `metrics.go` map: `{"deepseek-v4": {prompt: 0.002, completion: 0.008}, "gpt-4o-mini": {...}}` CNY per 1k token；未知 model fall back 到 `unknown` 标签并记 0 元（不 panic）。
4. router 新增 `/internal/v1/metrics` endpoint（用 `promhttp.Handler()`），仅 `INTERNAL_API_KEY` bearer 准入；其它 endpoint 无影响。
5. integration test：mock LLM 返回 `Usage{prompt_tokens: 100, completion_tokens: 50}`；test 跑后 `tally_llm_tokens_total{tenant="t1",model="deepseek-v4",direction="in"}` 增 100，`direction="out"` 增 50；`tally_llm_cost_cny_total` 增 `0.0002 + 0.0004 = 0.0006`。
6. `curl -H "Authorization: Bearer $INTERNAL_API_KEY" http://localhost:18200/internal/v1/metrics | grep '^tally_llm_'` 见两 metric。
7. 业务代码不直接 import `prometheus/client_golang` — 全经 `llmgateway` package；后续 ratelimit / sqlguard 同包扩展。

## Dev Notes

- **单价 source of truth 暂 hardcode**；未来 Hub 计费切流后改读 `LURUS_LLM_BASE_URL` 元数据。本 story 不做。
- counter 不要在每次请求 `Register` — 全局 `init()` 注册一次即可。
- tenant label 高基数风险 — 监控 cardinality；超 1000 个 tenant 需要改 histogram 抽样（V2 处理，本 story 不防御）。
- `RecordUsage` 必须 idempotent / 不 panic — LLM 解析失败时 in/out 可能为 0，照样调，0+0 没副作用。
- 别在 hot path 加 mutex；prometheus counter 内部用 atomic。
