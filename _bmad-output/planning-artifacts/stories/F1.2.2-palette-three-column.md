## Story

As 跨境老板 typing in ⌘K,
I want to see 3 distinct sections — "新建 PO / 跳 SKU / 帮我查" — rendered within 200ms of opening the palette,
so that I form muscle memory: ⌘K 就是我的"全局做事入口"，而不是又一个搜索框。

## Acceptance Criteria

1. 打开 ⌘K 到首屏可点击的时间 ≤ 200ms（Performance API `palette_open → first_paint` 实测; 10 次取 P95）。
2. 三栏标签固定顺序：**操作**（新建 PO / 新建销售单 / 新建商品）→ **跳转**（SKU / 客户 / 供应商 / 页面）→ **AI 帮我查**（≥5 字符后才出现，调 F1.2.1 NLQ）。
3. 当 query ≥ 5 字符且 NLQ 返回 `action=navigate|query` 时，AI 栏第一行展示自然语言意图（"查询：上月毛利 TOP 5 SKU"），按 Enter 直接路由；NLQ 失败/超时显示 "打开 AI 助手 →"（降级路径）。
4. 键盘导航：↑↓ 跨栏移动 / Tab 在栏头切换 / Enter 激活 / ⌘+1/2/3 直跳栏 / Esc 关闭。
5. 不阻塞输入 — NLQ 请求 debounce 250ms；用户继续打字时取消旧请求（AbortController）。
6. 现有 `Palette.test.tsx` 全绿 + 新增 4 个用例（三栏渲染顺序 / debounce / abort / NLQ 降级）。

## Dev Notes

- 200ms 预算严格：不允许在 palette open 时同步调任何 API；NLQ 永远是后置 debounce 触发
- 三栏 = 哲学定锚原则 ② "⌘K 是肌肉记忆,不是搜索框"的载体；评审看到"AI 栏放底部用户看不见"直接打回
- 不要把"跳转"栏做成模糊搜索 — 用户输 "sku-12345" 必须精确匹配 product code（这是 F1.2.3 的工作，本 story 可先 hardcode 假数据）
- 降级是体验红线 — 测试时强制超时跑一次, 不允许"hang 在那转圈"
