## Story

As 用户站在某个 SKU / 销售单 / 仪表盘页打开 AI Drawer，
I want Drawer 顶部展示一个"上下文 chip"告诉我 AI 知道我在看什么 + 给我 3 个针对该页面的 suggested prompts，
so that 我不用绞尽脑汁想问什么，按一下"这个 SKU 周转怎么样"就出答案；上下文不对时我也能一眼看见、点 X 清掉。

## Acceptance Criteria

1. Drawer 打开时通过 `usePageContext()` hook 拿到当前 route + entity（监听 Next.js `usePathname` + URL params），传给 `streamChat` 作为 `page_context` 字段。
2. Drawer 顶部 chip 显示 "📦 USB-C 数据线 1m" 或 "📊 仪表盘" 或 "📋 销售单 SO-2026-0123"，点 X 可清除（清除后 chat 不带 context, fallback 通用回答）。
3. 3 个 suggested prompts 按 entity_kind 模板：
   - product: 「这个 SKU 周转怎么样」「最近 30 天卖了多少」「需要补货吗」
   - sales_bill: 「这单毛利率」「客户历史购买记录」「同类商品销量对比」
   - dashboard: 「今天最该关注什么」「本周哪个 SKU 表现最好」「下周哪些会缺货」
   - 其他 / 无 entity: 通用 3 个 prompt（来自配置文件）。
4. 点 suggested prompt = 自动填入 input 框并立刻发送（不需要二次点发送）；走 `single_shot=true` 模式（F1.4.1 提供）。
5. ⌘K → "打开 AI 助手" 降级路径 (F1.2.2) 触发 `tally:ai-query` event 时，Drawer 必须**保留**当前 page_context（不要因 query 而忽略 context）。
6. 现有 `MessageList.tsx` / `PlanCard.tsx` 不动；本 story 仅扩 `Drawer.tsx` + 新 component `<ContextChip />` + `<PromptSuggestions />`。

- usePageContext: `/dashboard` → kind=dashboard, id=null

- `bun run test` 全绿
- `bun run lint` + `bun run build` 0 warning

### 手测
- 站在 `/products/<id>` → 开 Drawer → chip 显示 SKU 名 + 3 个 prompt
- 点 "这个 SKU 周转怎么样" → input 立填 → 自动发送 → AI 一句话答（< 2s 看到首字）
- 点 X 清掉 chip → 下个问题不带 context（dev tools network 看 payload 无 page_context）
- ⌘K → 输 "上月毛利 TOP" 5 字符以上 → Tab 进 AI → Enter 进 Drawer → chip 仍在 ✓

## Dev Notes

- ContextChip 不允许把"AI 在看什么"隐式化 — 用户必须 see-then-trust；UX 设计原则
- PromptSuggestions 行数严格 ≤ 3（4+ 个 = 用户麻痹，研究 B 提到）
- 不接 server-side prompt config — `prompt-templates.ts` 编译进 bundle；后期需要远程 config 再加（YAGNI）
- 测 Drawer 时 mock `streamChat` 避免打真后端
