# Story F1.2.3: ⌘K v3 — Entity 搜索 + Recent + 埋点 DAU 指标

Status: ready-for-dev

**Owner:** tally fullstack
**Repo:** `2b-svc-psi`
**Estimate:** 12h
**Blockers:** F1.2.2 (三栏 UI 就位)

## Story

As 跨境老板 输入 "sku-12345" 或 "深圳华强" 在 ⌘K，
I want 跳转栏精确返回该 SKU / 客户 / 供应商 entity 并按 Enter 直接到详情页，
so that 我 ≤ 2 keystroke 命中目标，不用过列表页二次点击；同时埋点能让我们 30 天后看清"⌘K DAU 渗透率 ≥ 60%"达没达。

## Acceptance Criteria

1. 新 endpoint `GET /api/v1/search/entities?q=&kind=&limit=10`，返回
   `{"items":[{"id","kind","label","href","subtitle"}]}`；kind ∈ `{product,customer,supplier,page}`；省略 kind = 全 kind 混搜。
2. 后端 SQL 用 `pg_trgm` GIN index（product.code/name + customer.name + supplier.name）；P95 ≤ 60ms（DB EXPLAIN 实测 + benchmark 1k entity 数据集）。
3. ⌘K 跳转栏调 entity search debounce 200ms，结果按 kind 分组渲染（emoji 前缀：📦 SKU / 👤 客户 / 🚚 供应商 / 📄 页面）。
4. **Recent 栏**：取浏览器 localStorage 最近 5 个点击的 entity，palette 打开默认显示（无 query 时）；Recent 栏顶部展示，跳转栏在它下面。
5. 埋点事件：
   - `palette_invocation`（每次 ⌘K 打开）
   - `palette_select`（property: `kind=action|jump|ai`, `keystrokes`, `latency_ms`）
   - `palette_dismiss`（property: `had_results`, `query_length`）
   3 个事件入现有埋点 sink（PostHog / 自建均可，确认环境后接），数据进 Postgres `tally.event_log` 表（已存在则复用，否则在本 story 加 m31 migration）。
6. 周报 SQL（可手跑） 输出"过去 7 天 ⌘K DAU 渗透率 = palette 用户数 / 总活跃用户数"；放 `doc/queries/palette-dau.sql`。
