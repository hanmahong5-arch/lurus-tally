## Story

As the product founder,
I want H1 / H2 / H3 三个 falsifiable 假设的 owner 邮箱栏 + deadline + falsification_threshold 全部签字非空,
so that 8 维度自评里"客户/市场验证 3.5"分项不再是"假设无 owner 跑成数据"，并且 Kill Switch 飞书告警（S10-Q）的 issue assignment 有自动收件人。

## Acceptance Criteria

1. `_bmad-output/planning-artifacts/assumptions.md` 三段 YAML 块的 `owner: TBD` 字段全部替换为真实邮箱（**founder 本人邮箱，不接受未来招的人**）。
2. `deadline:` 字段保留 `2026-W36 (Sprint 12 end ≈ 2026-08)` 或更紧（不放松）。
3. `falsification_threshold:` 三条不动；如 founder 认为某条阈值不合理，**先在本 story 修改并 commit message 说明 why**，再签字。
4. commit message 模板：
   ```
   docs(v1.5): S0-C1 founder sign H1/H2/H3 assumptions

   - H1 owner: <email>  threshold: <unchanged|new>
   - H2 owner: <email>  threshold: <unchanged|new>
   - H3 owner: <email>  threshold: <unchanged|new>

   Co-Authored-By: <founder name> <founder@email>
   ```
5. Founder 在本 story Dev Agent Record 段加一条 "Signed: <name> <date>" 作为人手 attestation。
6. S0-C2 (`bin/assumption-snapshot.sh`) 实施前**必须先**通过本 story；snapshot 脚本读 owner 邮箱用来后续 KS alert email subject。
