# Story F1.2.1: ⌘K v3 — NLQ Backend (`POST /api/v1/ai/nlq`)

**Owner:** tally backend

**Blockers:** 无（复用现有 `internal/app/ai/orchestrator.go` + `llmclient`）

## Story

As a user typing in ⌘K palette,
I want a backend endpoint that turns "上月哪 5 个 SKU 毛利最高" into a structured `{action, target, args}` JSON,
so that the palette can route my natural-language query to navigate / query / execute without falling into a free-form chat black box.

## Acceptance Criteria

1. New endpoint `POST /api/v1/ai/nlq` returns `200 OK` with body
   `{"action":"navigate|query|execute","target":"<route|tool>","args":{...},"confidence":0.0-1.0,"fallback_reason":"<string|null>"}`.
2. LLM call MUST use `response_format: json_schema` (strict) — schema匹配 above structure; non-conforming reply → handler returns 502 + `fallback_reason="schema_violation"`.
3. `execute` action 永远不直接落库 — handler 调用 orchestrator Plan-pending 路径，response 里 `args.plan_id` 指向 `/api/v1/ai/plans/:id`.
4. Request budget: P95 ≤ 800ms (LLM call P95 600ms + 200ms 服务端开销)；超 3000ms 服务端强制返回 `{"action":"navigate","target":"/ai-drawer","args":{"query":<原文>},"fallback_reason":"timeout"}` (降级到 Drawer)。
5. 写一条 `audit_log` (`source='palette'`, `model`, `tokens_in/out`, `latency_ms`)；不写就视为故障。
6. 单元测试覆盖：schema-valid 路径 / schema-violation / timeout / execute → plan 路径；mock LLM 不打真模型。

## Dev Notes

- 字符触发不在本 story — palette FE 已有 `AI_TRIGGER_MIN_CHARS=5`；F1.2.2 才接 NLQ
- prompt 不要 few-shot 塞太多 example（token 浪费）；走 strict schema 让 model 自己学
- 失败时一定降级，不阻塞用户 — "你看到 ⌘K 卡 3 秒"是 F1.2 致命体验 bug
- Plan-pending 路径绝不能在 NLQ handler 直接执行 destructive op — 见架构红线 #4
