## Story

As an assumption tracker,
I want a daily `bin/assumption-snapshot.sh` that scrapes Prometheus + reads manual fields → writes today's `current_value` + `last_evidence_url` into `assumptions.md`,
so that H1/H2/H3 deadline approach 时 founder 能从一个 YAML 看到过去 90d 走势，而不是"上次看是 1 个月前不知道现在怎样"。

## Acceptance Criteria

1. 新建 `bin/assumption-snapshot.sh`（POSIX bash，shellcheck 绿）：
   - 读 `_bmad-output/planning-artifacts/assumptions.md`
   - 对 H1：scrape `tally_trial_conversion_d90` from Prom（如 metric 不存在 → set value `n/a (metric absent)`）
   - 对 H2：从 `_bmad-output/planning-artifacts/retail_pilot_log.md`（如不存在 → `n/a (no retail pilots yet)`）grep `tally_ocr_invoice_recorded_total` + `tally_feishu_dingtalk_confirm_total` from Prom
   - 对 H3：计算 `tally_palette_invocation_dau / tally_total_dau` + `tally_ai_drawer_open_dau / tally_total_dau`（Prom recording rule，如未 ready → `n/a`）
2. 写入方式：**append 一行**到 assumptions.md 末尾 "状态历史" 表，格式 `| YYYY-MM-DD | <H1 status> | <H1 value> | <H2 status> | <H2 value> | <H3 status> | <H3 value> |`；不修改 H1/H2/H3 YAML 块（避免 owner 签字丢）。
3. status 决策逻辑（脚本内）：
   - value 命中 falsification_threshold → `falsified`
   - value 未达但仍在 trial 期内 → `pending`
   - value 满足（≥阈值）→ `truthy`
   - value `n/a` → `inconclusive`
4. cron 注册：在 governance repo `lurus/.github/workflows/` 或 R6 主机 crontab `0 8 * * *`（北京时间 16:00，给前一日数据充裕缓冲）；脚本 exit code 非 0 时发飞书 webhook `ASSUMPTION_SNAPSHOT_FAIL_FEISHU`。
5. 90d trim：脚本末尾保留最近 90 行（`tail -90`），更老的归档 `_bmad-output/planning-artifacts/_archive/assumption-history-<YYYY-Q>.md`。
6. Commit 后跑一次手动验证：`bash bin/assumption-snapshot.sh` 在 assumptions.md 末尾追加一行（即使全 n/a 也写）；diff 在 PR 里可见。

## Dev Notes

- **n/a 不是失败** — 头 8 个 sprint 三个 H 大概率全 n/a；脚本"工作正常"= append 一行写明 n/a。这本身就是 evidence_gap_days 信号源（S0-C3 health-report.sh 用）。
- **不要用 yq / Python** — bin/ 脚本只 bash + curl + jq + awk，跨 win/mac/linux dev box 一致。
- **cron 跑在哪**：推 GH Actions（governance repo） — R6 主机 crontab 可被磁盘重装、网络中断破坏，GH Actions schedule 更稳。
- 脚本失败别静默 → 发飞书 `ASSUMPTION_SNAPSHOT_FAIL_FEISHU` webhook（在 K8s secret 注入）。
- `last_evidence_url` 暂留空 — 客户访谈录音 link 由 founder 手填到 H1/H2/H3 YAML，脚本不动该字段。
