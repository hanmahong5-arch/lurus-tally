## 新增 Epic 草案 (E21-E27)

### Epic 21 — 可恢复性框架（**Top priority**）
**目标**: 用户绝不丢工作。任何崩溃/误操作/网络抖动 30 秒内可还原。
- S21.1 IndexedDB 草稿层：所有 form 自动持久化，区分"已同步 / 仅本地"标记
- S21.2 全局 Cmd+Z 撤销栈：删除 / 改价 / 状态切换 30s 窗口可撤销（toast 模式）
- S21.3 单据时间线：右侧 trail 显示 who/when/what 字段级 diff
- S21.4 "我做到哪了" 卡片：首页 sticky，未完成草稿 + 24h 异常 + 今日待确认
- S21.5 跨设备草稿同步（可选，依赖 S21.1）

**预估**: 60h / 2 sprint
**依赖**: E1-E7 已交付（数据模型稳定）
**收益**: 全站通吃；用户感知最强的"安心感"

### Epic 22 — 性能基线
- S22.1 商品/单据虚拟滚动（react-window）+ FlexSearch 客户端索引
- S22.2 路由 hover 预取（Next.js Link prefetch tuning）
- S22.3 RSC + Suspense 流式渲染，首屏 TTI < 500ms
- S22.4 Web Vitals 上报到 OTel + 仪表盘

**预估**: 40h / 1.5 sprint
**触发条件**: 单租户 SKU > 5k 时启动

### Epic 23 — 主动 AI 推送
> 与 E17 (Kova 补货 Agent) 的差异：E17 是 pull-mode "用户问"；本 epic 是 push-mode "AI 主动告诉"
- S23.1 断货预警引擎：销售速度 + 安全库存 → cron 扫描 → AI Drawer 通知
- S23.2 异常检测：退货率 / 客单价 / 毛利率突变 (Z-score > 2σ) 推送
- S23.3 周报自动生成：周一 8 点用 AI 生成 "本周经营摘要" 推送邮件 + 站内
- S23.4 用户阈值与置信度调节面板（误报治理）
- S23.5 OCR 单据录入：上传发票/送货单 → AI 抽取 → 一键转入库

**预估**: 80h / 3 sprint
**依赖**: E10 (销售流程) + AI Drawer (已交付)
**风险**: 误报快速失信，必须保守阈值 + 用户可调

### Epic 24 — 协作与权限
- S24.1 角色矩阵：销售 / 采购 / 财务 / 老板 / 自定义
- S24.2 审批流：大单 / 退款 / 调价超阈值（DSL 配置 if-then）
- S24.3 单据 @某人 + 评论 + 邮件/微信通知
- S24.4 实时编辑冲突提示（"张三 30s 前改过此单"）

**预估**: 70h / 3 sprint
**依赖**: E2 (RLS) + E11 (Onboarding 双 persona)
**触发条件**: 客户团队规模 > 3 人

### Epic 25 — 输入 / 输出生态
- S25.1 扫码枪原生（USB/蓝牙 HID, WebHID API）
- S25.2 打印机适配：面单 / 标签 / 小票（佳博 / 小白盒 / Bambu）
- S25.3 微信 / 钉钉机器人：单据推送 + 审批回应（webhook 双向）
- S25.4 数据导出全集：CSV / Excel / JSON / API token
- S25.5 开放 API（OpenAPI 3.1 + 限流 + 配额）

**预估**: 90h / 3 sprint
**依赖**: E6/E7 流程闭环
**收益**: 实店落地必需；同时降低用户离开成本（信任锚）

### Epic 26 — 信任与可观测
- S26.1 完整操作审计日志（who/what/when，全字段 diff，可导出）
- S26.2 数据健康度评分（商品/客户/供应商资料完整度）
- S26.3 变更日志面板（系统迭代可见，"本周新增 X / 改进 Y"）
- S26.4 GDPR/PIPL 数据导出 + 删除请求

**预估**: 50h / 2 sprint
**依赖**: 全部业务表
**收益**: 老板视角刚需（查岗）+ 合规

### Epic 27 — 深度业务（按客户付费意愿排序选做）
> 不全做。先看哪类客户付费 → 选 1-2
- S27.1 多仓库 + 调拨单（连锁刚需）
- S27.2 批次 / 序列号追溯（食品/医药）
- S27.3 BOM + 半成品 + 委外加工（制造业）
- S27.4 电商平台同步（淘宝/拼多多/抖店）
- S27.5 多币种 + 实时汇率（跨境，部分已在 E9）

**预估**: 每子项 60-100h
**触发条件**: 有付费客户明确诉求

## 决策原则

1. **可恢复性 > 新功能** — 用户丢一次工作的负面口碑 = 10 个新功能
2. **主动 AI > 被动 AI** — 用户不会主动问，要 AI 主动开口
3. **生态 > 深度** — 扫码枪 + 微信机器人比多仓库更普适
4. **信任 > 锁定** — 数据导出做得好，反而留得住
5. **性能不是优化是设计** — 第一行代码就考虑虚拟化和预取
